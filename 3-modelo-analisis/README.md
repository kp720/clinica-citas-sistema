# 3. Modelo de análisis

**a. Diagrama de caso de usos**
- El diagrama de **casos de uso** representa un sistema de **Farmacia con Consultorio** y muestra la interacción entre tres actores principales:


i. **Encargado de farmacia**:

    - Inicia sesión en el sistema, registra pacientes y citas.

ii. **Paciente**:

    - Solicita una cita.

iii. **Médico**:

    - Consulta la agenda, verifica citas y actualiza o crea el historial
      médico del paciente.
      
<img src="./media/image1.png" style="width:6.33333in;height:4.65071in" />

- Los casos de uso están interconectados para mostrar el flujo de
actividades, desde la solicitud de una cita hasta la gestión del
historial médico, con roles bien definidos para cada actor.

**b. Diagrama de secuencias:** 
- El diagrama de secuencia ilustra el
flujo de interacción entre los actores y el sistema en el contexto de un
consultorio médico. En este caso, los actores principales (Paciente,
Doctor, cita, Encargado y expediente) participan en un flujo manual
donde se destacan procesos como la solicitud de citas, las consultas
médicas y el manejo de expedientes.

- El Encargado registra las citas del paciente y se comunica con el Doctor
para confirmar disponibilidad, mientras que el Doctor gestiona los
expedientes médicos y lleva a cabo las consultas correspondientes. Este
diagrama resalta cómo los pasos del proceso se desarrollan de manera
cronológica.
